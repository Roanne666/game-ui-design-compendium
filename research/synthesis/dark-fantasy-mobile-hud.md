## 1. 一句话定位

**世界在中间，拇指在两侧；常驻控件用生成暗金图标，文案只出现在「世界提示 / 石板菜单 / 无障碍标签」里。**

魂系 UI 的核心不是「好看的框」，而是：**能消失、有材质、不抢画面、不长得像网站按钮。**

---

## 2. 设计支柱（FromSoft 思路 → 手机转译）

| 支柱 | PC 魂系习惯 | 竖屏触控转译 |
|------|-------------|--------------|
| 沉浸优先 | HUD 极简、情境浮现 | 生命/集中/耐力**默认淡出**，变化后约 2s 再隐 |
| 世界给线索 | 赐福光、篝火声、敌人动作 | 地名弱化；近篝火才出休息提示；中央提示框情境出现 |
| 菜单像古物 | 石板 / 羊皮纸 | 篝火中央石板（休息 / 离开），双层古金边 + 菱形饰 |
| 常驻操作 | 手柄键位，几乎无屏幕字 | **必须有触控键**，但用**图标**而非汉字标签 |
| 克制装饰 | 少即是多 | 双层边 + 角饰 + 金属渐变即可；禁止霓虹、扫描线、大面积辉光 |

参考阅读（外部）: Elden Ring 极简 HUD 争论、FromSoft「情境浮现」论述；与本仓库风格文档互证，**不照搬商业素材**。

## 3. 竖屏信息架构（9:20）

把屏幕切成互不抢位的层。Demo 踩过的坑：**帮助文、露滴、生命条、右键簇挤在同一高度**。

```
┌─────────────────────────┐
│  地名（弱）        [?]  │  ← 顶：信息 / 可选指引入口
│  （近篝火时：休息提示）   │
│                         │
│      【情境提示框】       │  ← 中：仅需要时出现
│                         │
│                         │
│  ┌生命条┐               │  ← 中下左：短 vitals（可隐）
│  │      │   [露滴]   [互]│  ← 中下：露滴居中；右列次要键
│  │      │           [滚]│
│ [锁定]              [攻] │  ← 底：拇指主操作
└─────────────────────────┘
```

### 3.1 分区规则

| 区 | 内容 | 规则 |
|----|------|------|
| 顶中 | 地点名 | 低对比骨白；letter-spacing 拉开；不抢画面 |
| 顶右 | 帮助 `?` | **默认不展开**；指引文上移到顶区，禁止压底栏 |
| 中央 | 情境提示 | 点按「互动」· 篝火 等；双线饰 + 暗底；无操作时隐藏 |
| 中下左 | 生命 / 集中 / 耐力 | 窄列（约 ≤34% 宽），**勿与露滴水平重叠** |
| 中下中 | 露滴瓶 | 居中；仅图标 + 角标数量 |
| 右下竖列 | 互动 → 翻滚 → 攻击 | **叠在攻击正上方**，把中央让给露滴 |
| 左下 | 锁定（按住） | 圆形大热区 |
| 中央模态 | 篝火石板 | 盖住中景；可点休息 / 离开 |

### 3.2 安全间距（经验值）

- 触控主键热区 ≥ 48 CSS px（攻击 / 锁定建议 ≥ 68）
- 露滴与右列键：水平不得抢同一带；右列贴右侧堆叠
- 帮助展开时 `top` 落在地名下方，**永不** `bottom: ~100px` 与露滴同带

## 4. 控件哲学：图标生成，不是网页字钮

### 4.1 为什么不要汉字贴在键上

竖屏上「锁定 / 互动 / 翻滚 / 攻击」四个字钮会立刻读成 **HTML 表单 / 落地页 CTA**：

- 字重、字距、矩形金边 ≈ Web 按钮组件
- 中文标签占满热区，材质感被文字吃掉
- 与魂系「手柄隐喻」冲突：玩家认的是**形**，不是按钮文案

### 4.2 正确做法

| 控件 | 可见层 | 无障碍 / 学习 |
|------|--------|----------------|
| 锁定 / 互动 / 翻滚 / 攻击 | **生成暗金金属图标** | `aria-label` 中文 |
| 露滴 | 生成瓶形图标 + 角标数字 | `aria-label="黄金种子露滴"` |
| 帮助 | 小圆键 + 生成问号图标 | 点开后中文指引 |
| 篝火石板 | **保留中文**「休息 / 离开」 | 菜单允许文字（古书隐喻） |
| 情境提示 / Feed | 中文短句 | 世界内文案，不是 chrome |

### 4.3 图标资产规范（本 Demo）

路径: `demo/scenes/02-dark-fantasy/art/icons/`

| 文件 | 语义 |
|------|------|
| `icon-lock.png` | 锁定准星 / 十字圣徽 |
| `icon-use.png` | 互动手势 |
| `icon-roll.png` | 翻滚 / 闪避动势 |
| `icon-attack.png` | 长剑 |
| `icon-estus.png` | 治愈瓶 |
| `icon-help.png` | 指引 |

制作约定:

1. **项目原创生成**，暗金 / 青铜浮雕，黑底 → 抠透明（见 `demo/assets/CREDITS.md`）
2. 导出约 160×160，HUD 内显示 28–38 px，靠 `drop-shadow` 压进金属底
3. 禁止直接贴 Kenney 彩色扁平图标当魂系主键（易回网页感）
4. 禁止商业魂类 / 官方 Rip 图标

## 5. 视觉配方（克制魂系）

### 5.1 色（与 Spec 一致）

- 古金 `#C9A961` / 暗金 `#8B7332` / 骨白 `#E8DCC4` / 近黑 `#0A0908` / 深棕面板 `#1F1611`
- 低血 `#8B0000`；集中条偏冷石青；耐力跟古金

### 5.2 容器

- **双层边**: 外 1–2px 古金 + 内 inset 暗金线
- **角饰**: 四角 L 形短线（或菱形 ◆ 用于石板顶底）
- **金属底**: 深棕斜向渐变 + 顶部微高光 + 底部内阴影（内凹锻造感）
- **按下**: `scale(0.94)` + 暗金铺底 + 边变亮；时长约 100–140ms ease-out

### 5.3 刻意不做

- ❌ 常驻大段操作说明压在底栏
- ❌ 触控键上排汉字 / 英文胶囊标签
- ❌ 共享 `components.css` 网页卡片壳当最终 HUD
- ❌ 霓虹、扫描线、大面积 glow
- ❌ 把横屏四角 PC HUD 原样塞进 9:20

---

## 6. 文案语言

- 玩家可见：**一律中文**（地名、提示、Feed、石板、帮助）
- 专有名词可括注，但主文案仍是中文
- 帮助文用触控语义：「点按 / 拖动 / 按住」，不写键位表

---

## 7. 玩法闭环（验收清单）

不看帮助文也应能完成:

1. 拖动环视废墟
2. 点敌人或按住锁定 → 攻击 / 翻滚耗耐力
3. 受击掉血；低血条变红；点露滴回血（有限次）
4. 靠近篝火 → 情境提示 → 互动 → 石板「休息」回满并补露滴

| 项 | 路径 |

| 早期设计确认稿 | `docs/superpowers/specs/2026-07-24-dark-fantasy-mobile-hud-design.md` |

技术备忘:

- 场景自包含 CSS；**不要**为了省事 `@import` 共享 web components 当最终观感
- 生命条用 `opacity` 显隐，而不是占布局把底栏顶乱
- 图标用 `<img>` + `aria-label`，键面 `font-size: 0` 避免残留字偶发露出

## 9. 可复用到其它「重氛围动作 RPG」的检查表

做下一套类似 HUD 时，先问:

1. 常驻键是不是**图标 / 物件**，而不是网页字钮？
2. 底栏有没有**明确分带**（左锁定 / 中道具 / 右攻击簇）？
3. 帮助与 Feed 是否离开拇指区？
4. 数值条是否默认隐、变化才显？
5. 菜单是否像石板/古书，而战斗中 HUD 仍克制？
6. 文案是否中文且触控语义？
7. 美术是否项目原创或开放许可，无商业 Rip？

全部「是」才算达到本参考的完成度。

---

## 10. 反例（Demo 踩坑摘要）

1. 帮助文 `bottom: 108px` + 露滴同带 → **必遮挡**
2. 互动/翻滚放在攻击左侧 → 与居中露滴 **水平打架**
3. 生命条宽度 44% + 居中露滴 → **叠字叠框**
4. 键上写「攻击」等 → **网页感回潮**
5. 线框 SVG 当最终图标 → 比字钮好，但仍偏「UI kit」；**生成金属浮雕**更贴世界
