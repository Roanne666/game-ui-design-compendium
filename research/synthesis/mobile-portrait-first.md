# 移动端竖屏优先 — 通用 HUD 思路

> 本仓库**所有风格**的默认交付：竖屏游戏 UI。桌面浏览器仅作预览壳。  
> 本文 = **硬性准则** + **从 01/02 可玩场景迭代沉淀的通用落地思路**。  
> 游戏感 vs 网页见 [`game-feel-vs-web.md`](./game-feel-vs-web.md)。  
> Style 02 案例附录见 [`dark-fantasy-mobile-hud.md`](./dark-fantasy-mobile-hud.md)。

日期: 2026-07-24

---

## 0. 一句话

> **世界在中间，拇指在两侧；常驻操作认图标/物件，文案留给情境提示与菜单——先为拇指设计，再为预览兼容键鼠。**

---

## 1. 目标形态

| 项 | 约定 |
|----|------|
| 主画幅 | **竖屏**，默认参考比 **9:20**（亦兼容 9:16–9:19.5） |
| 主输入 | **触控**（点按、拖拽、长按、双指可选） |
| 预览 | 桌面用居中手机框模拟；**不得**把键鼠操作写成主操作说明 |
| 拇指区 | 关键操作落在屏幕下半区左右拇指可达范围 |
| 文案 | 玩家可见 UI **一律中文**；`aria-label` 同步中文 |

---

## 2. 必须遵守（硬规则）

1. **布局按竖屏信息架构**：顶状态 / 中视野 / 底操作栏；禁止把横屏四角 PC HUD 原样塞进竖屏。
2. **操作可单手或双手拇指完成**：主操作须有 **可见触控控件**（热区 ≥44×44 CSS px，推荐 ≥48；主键建议 ≥68）。
3. **文案与提示用触控语义**：写「点按 / 拖动 / 长按」，不写「LMB / RMB / WASD / Tab」作为主路径。
4. **避免悬停依赖**：`:hover` 仅作增强；状态反馈以 `active` / `pressed` / 选中为准。
5. **安全区**：顶/底为刘海与手势条留白；关键数值勿贴物理边缘。
6. **皮肤/方言切换不得以键鼠为前提**：切换入口必须是屏幕上的按钮。
7. **Demo 验收**：真机或 DevTools 竖屏下，不看帮助文也能完成核心循环。
8. **界面文案一律中文**：按钮含义、标签、任务、提示、日志、菜单；专有名词可括注原文。

---

## 3. 禁止（键鼠与网页惯性）

- ❌ 把右键、中键、键盘快捷键当作**唯一**操作方式
- ❌ 帮助条主文案是键位表（`LMB FIRE · RMB LOCK · WASD`）
- ❌ 桌面全屏铺满当作默认交付形态（竖屏手机框才是默认）
- ❌ 控件热区过小、挤在屏幕上沿中央（竖屏难够）
- ❌ 依赖鼠标精确瞄准而不提供触控瞄准/锁定辅助
- ❌ **常驻触控键上排汉字/英文胶囊**当主键标签（极易读成网页 CTA）
- ❌ 常驻大段操作说明压在底栏拇指区
- ❌ 用共享网页组件壳（卡片、圆角 pill、Inter 风）冒充最终游戏 HUD

键盘快捷键若保留，仅作桌面调试捷径，**不得**出现在玩家主帮助里。

## 4. 通用竖屏分层（所有类型共用）

把 9:20 画幅切成**互不抢位**的层。核心教训：同一高度塞「帮助文 + 道具格 + 生命条 + 右键簇」必遮挡。

```
┌─────────────────────────┐
│  状态/地名（弱）    [?] │  ← 顶：信息 / 可选指引入口
│                         │
│      【情境提示】         │  ← 中：仅需要时出现
│       （游戏世界）        │
│                         │
│  ┌短状态┐               │  ← 中下左：可隐 vitals / 资源
│  │      │  [道具]   [次]│  ← 中下：道具居中；次要键贴右竖排
│  │      │           [次]│
│ [辅键]              [主] │  ← 底：拇指主/辅操作
└─────────────────────────┘
```

### 4.1 分区通则

| 区 | 放什么 | 通则 |
|----|--------|------|
| 顶 | 地名、时钟、弱状态、帮助入口 | 低对比；帮助**默认收起**；展开文案留在顶区 |
| 中 | 世界画面 + 情境提示 | 提示框可出现可消失；不常驻教程式长文 |
| 中下左 | 短资源条 / 轻量状态 | **限宽**（经验 ≤34%），勿与居中道具格重叠 |
| 中下中 | 消耗品 / 快捷道具 | 图标 + 角标数量；不写满格汉字 |
| 右下 | 主操作 + 其上叠放次要操作 | **竖列贴右**，把中央让给道具 |
| 左下 | 辅操作（锁定、冲刺、瞄准等） | 圆形或大方热区；按住类用 `pressed` |
| 中央模态 | 休息点 / 暂停 / 确认 | 允许文字菜单；盖住中景即可 |

### 4.2 间距经验

- 主键热区 ≥ 48px（攻击/开火/确认类建议 ≥ 68）
- 居中道具与右列键：**水平分带**；右列叠在主键正上方，勿伸进中央
- 帮助展开用 `top`（地名下），**禁止**与底栏同用 `bottom: ~100px` 一带
- 可隐状态条用 `opacity` 显隐，避免撑乱底栏布局

### 4.3 按类型微调（同一骨架）

| 类型倾向 | 左下 | 右下主键 | 中下道具 | 顶 |
|----------|------|----------|----------|----|
| 魂类 / 动作 RPG | 锁定（按住） | 攻击；上叠翻滚/互动 | 治愈瓶等 | 地名弱化 |
| 赛博潜入 | 冲刺（按住） | 开火；上叠互动/黑客 | 武器口袋 | 方言/扫描状态 |
| 射击 / 战术 | 姿态/掩体 | 开火；上叠换弹 | 弹药/技能 | 罗盘/任务短讯 |
| 菜单主导（JRPG 等） | 可弱化战斗键 | 确认/取消 | 可无 | 章节/菜单栈 |

骨架不变：**中空、底拇指、顶信息、右主左辅。**

## 5. 控件哲学：图标 / 物件，不是网页字钮

### 5.1 问题

竖屏常驻键若写满「攻击 / 开火 / 互动」，会立刻读成 **HTML 按钮 / 落地页 CTA**：

- 字重 + 字距 + 描边矩形 ≈ Web 组件
- 文字吃掉材质与风格辨识
- 与「手柄/实物隐喻」冲突：玩家认的是**形**

### 5.2 通则

| 层 | 可见 | 说明 |
|----|------|------|
| 常驻触控键 | **风格化图标或物件**（优先项目生成，贴合该风格材质） | `aria-label` 中文；键面不排主键汉字 |
| 快捷道具格 | 图标 + 角标数字 | 同左 |
| 帮助入口 | 小图标键 | 点开后才出中文指引 |
| 模态菜单 / 石板 / 日志 | **允许中文** | 阅读型界面，不是拇指 chrome |
| 情境提示 / Feed / 任务 | **中文短句** | 世界内文案 |

### 5.3 图标资产通则

1. **按风格生成或选用开放许可**，禁止商业游戏官方/Rip
2. 黑底/绿幕需抠透明；HUD 显示约 28–40px，源图可 128–160px
3. 扁平彩色「素材站通用图标」作主键时，易回网页感——优先**该风格材质**（暗金浮雕、霓虹线框、像素块等）
4. 线框 SVG 可作中间稿；终稿宜有材质，与世界板一致

## 6. 情境 HUD 与帮助

- **能隐则隐**：资源条可在数值变化时浮现，数秒后淡出（沉浸向类型尤甚）
- **情境提示**：靠近可互动物、选中目标时再出；无操作则隐藏
- **帮助**：默认关；入口放顶角；正文不上底栏
- **Feed / Toast**：短、中文、顶中或顶下，勿与拇指键重叠

---

## 7. 推荐触控映射（动作 / RPG 类）

| 意图 | 触控 |
|------|------|
| 环视 / 轻微视角 | 在世界区域 **拖动** |
| 选择目标 | **点按** 实体 |
| 主操作（攻/开火） | 右下 **大键**（图标） |
| 互动 / 次要技能 | 主键上方的 **次要键** 或情境提示后点按 |
| 冲刺 / 锁定 | 左下 **按住** |
| 换弹 / 扫描 / 日志 | 底栏或右列次级图标键 |
| 切武器 / 道具 | 底栏口袋格（点按） |

## 8. 文案语言

- 玩家可见一律中文
- 帮助与提示用触控语义
- 专有名词可括注原文，主文案仍是中文
- 无障碍：图标键必须有中文 `aria-label` / 等效可读名

## 9. 实现备忘（全风格）

- 场景**自包含**最终观感 CSS；可用 Token，但勿把网页 `components` 壳当验收标准
- 图标用 `<img>`（或等价）+ `aria-label`；键面避免残留可见字
- 按下反馈：`scale` + 材质态变化，时长按风格 Spec（魂系偏慢，赛博可更快）
- 新场景 / 大改默认交付：**9:20 + 触控 + 中文 + 本分层**

参考实现：

- Style 01：`demo/scenes/01-cyberpunk-hud/`（`layout-9x20.css` + 触控层）
- Style 02：`demo/scenes/02-dark-fantasy/`（分层 + 生成图标案例）

## 10. 验收检查表（通用）

1. 真机/竖屏 DevTools 下，不看帮助能否走完核心循环？
2. 常驻键是不是**图标/物件**，而不是网页字钮？
3. 底栏是否**分带**（左辅 / 中道具 / 右主+竖列次要）？
4. 帮助与 Feed 是否离开拇指区？
5. 有无横屏四角 HUD 硬塞、键位表主帮助、商业 Rip 图？
6. 玩家可见文案是否中文且触控语义？

全部「是」才算达到本仓库竖屏 HUD 完成度。

## 11. 与风格 Spec 的关系

- 各 `styles/*.md` 管**风格参数**（色、边、动效、反例）
