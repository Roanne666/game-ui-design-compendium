# 移动端竖屏优先 — 按游戏类型选布局

> 本仓库**所有风格**的默认交付：竖屏 **9:20** 游戏 UI。桌面浏览器仅作预览壳。  
> **画幅统一；楼型与玩法循环按游戏类型区分。** 禁止把「左锁定 + 右开火 + 中口袋」当成全库默认模板。  
> 相关：[`game-feel-vs-web.md`](./game-feel-vs-web.md)。

日期: 2026-07-24

---

## 0. 一句话

> **9:20 竖屏定死；世界在画面里；楼型与玩法跟游戏类型走——常驻认形（图标/仪表/指令窗），文案留给情境与菜单。**

---

## 1. 目标形态

| 项 | 约定 |
|----|------|
| 主画幅 | **竖屏 9:20**（硬规则；亦兼容预览时的 9:16–9:19.5 安全区） |
| 主输入 | **触控**（点按、拖拽、长按、双指可选） |
| 预览 | 桌面用居中手机框；**不得**把键鼠写成主操作说明 |
| 楼型 | **由游戏类型决定**（见 §4 矩阵）；不强制统一拇指动作条 |
| 文案 | 玩家可见 UI **一律中文**；`aria-label` 同步中文 |

---

## 2. 必须遵守（硬规则）

1. **画幅**：Demo `#viewport` 默认 `aspect-ratio: 9 / 20`；禁止横屏交付当主场景。
2. **类型正确的信息架构**：打开场景应能辨认游戏大类（魂 / 赛博 / 舰桥仪表 / 指令窗 / 菜单栈…），禁止「换皮同一套锁定开火」。
3. **操作可触控完成**：主操作须有可见控件（热区 ≥44×44 CSS px，推荐 ≥48）。
4. **文案用触控语义**：写「点按 / 拖动 / 长按」，不写「LMB / RMB / WASD / Tab」作主路径。
5. **避免悬停依赖**：`:hover` 仅增强；以 `active` / `pressed` / 选中为准。
6. **安全区**：顶/底为刘海与手势条留白。
7. **皮肤/方言切换**须有屏幕按钮，不得以键鼠为前提。
8. **盲测**：隐藏帮助后仍能完成该风格**矩阵规定的核心玩法循环**。
9. **界面文案一律中文**。

---

## 3. 禁止

- ❌ 把「左辅圆键 + 右主开火 + 中口袋」套到**不需要**该玩法的类型上（太空仪表、JRPG 菜单、像素指令窗、恐怖稀疏 HUD、街机标题等）
- ❌ 右键/键盘作为**唯一**操作方式
- ❌ 帮助条主文案是键位表
- ❌ 桌面全屏铺满当作默认交付
- ❌ 常驻触控键上排汉字/英文胶囊当主键标签
- ❌ 用网页卡片 / pill / Inter 壳冒充最终游戏 HUD
- ❌ 商业游戏官方 / Rip 素材

键盘捷径若保留，仅桌面调试，**不得**进玩家主帮助。

## 4. 权威矩阵：类型 → 布局 + 玩法

画幅一律 9:20。下表是楼型与玩法的**唯一默认**；风格 Spec 的「适用游戏类型」必须与此对齐。

| # | 风格 | 类型 | 布局原型 | 核心玩法循环（须可盲测） |
|---|------|------|----------|-------------------------|
| 01 | Cyberpunk | 赛博 RPG/潜入 | 顶任务+小地图；中世界；底武器口袋；左右冲刺/开火/黑客 | 扫描 → 标记 → 口袋切武器 → 开火/黑客 |
| 02 | Dark Fantasy | 魂类 | 顶弱地名；中空；底极少键 + 露滴侧置 | 靠近篝火休息；锁定；翻滚/攻击；喝露滴 |
| 03 | Sci-Fi Space | 太空模拟 | **仪表型**：上航向/系统；中大瞄准环+目标卡；**下半能量分配台**；导航纸片 | **调能量**（总和守恒）→ 点环锁定进度 → 武器能量够才可射击；导航改模式 |
| 04 | Pixel Retro | 2D RPG | 上半战场；**下半指令窗** | 点指令：攻击/技能/道具/逃跑 |
| 05 | Hand-drawn | Roguelike 叙事 | 中部三选一恩赐卡；底确认 | 选一张卡带走 |
| 06 | Cartoon Comic | 射击/节奏 | 顶连击；底**单一大拍键** | 点按开火/打拍，无锁定键 |
| 07 | Survival Horror | 恐怖 | 几乎无常驻条；底一枚背包入口 | 打开 spine/背包管理；世界内探索 |
| 08 | Anime Colorful | JRPG | **半屏/全屏菜单栈**；战斗时底指令条 | 菜单导航；战斗选指令 |
| 09 | Tactical Military | 战术 | 顶罗盘短讯；角小雷达；姿态/掩体 + 开火分区 | 切姿态 → 瞄准辅助 → 开火 |
| 10 | Synthwave Neon | 街机 | 标题层 → 一局；HUD 几乎只有分数 | 开始 → 得分 → 结束重开 |

验收：任意两套并排，**一眼看出玩法不同**，不能只靠换色/换图标。

### 4.1 避让与安全（跨类型）

- 同一高度勿堆「帮助长文 + 道具 + 生命 + 主键」
- 帮助默认收起，入口顶角，展开文案留在顶区
- Feed / Toast 短、中文，勿压主操作热区
- 动作类若用左右拇指键，须水平分带，避免互挡

### 4.2 「动作拇指条」何时可用

仅当矩阵玩法本身是动作射击/魂类/战术时（01 / 02 / 06 / 09 等）。  
**不是**默认骨架；03 / 04 / 05 / 07 / 08 / 10 禁止为了「统一」硬套锁定开火双圆键。

## 5. 控件哲学：认形，不是网页字钮

| 层 | 可见 | 说明 |
|----|------|------|
| 常驻触控 | **风格化图标、物件、仪表格、指令条** | `aria-label` 中文；主键面少排汉字 |
| 快捷道具格 | 图标 + 角标 | 动作 RPG / 赛博等需要时 |
| 帮助入口 | 小图标 | 点开后出中文指引 |
| 模态菜单 / 石板 / 日志 / 指令窗 | **允许中文** | 阅读与选择型界面 |
| 情境提示 / Feed / 任务 | **中文短句** | 世界内文案 |

图标：按风格生成或开放许可；禁止商业 Rip；黑底/绿幕需抠透明。

---

## 6. 情境 HUD 与帮助

- 能隐则隐（魂类/恐怖尤甚）
- 情境提示按需出现
- 帮助默认关；入口顶角
- Feed 勿与该类型主操作区重叠

---

## 7. 文案语言

- 玩家可见一律中文；触控语义；专有名词可括注原文
- 图标键须有中文 `aria-label`

---

## 8. 实现备忘

- 场景自包含最终观感 CSS；Token 可复用，网页 `components` 壳不作验收标准
- `#viewport { aspect-ratio: 9 / 20; }`
- 新场景对照 §4 矩阵选楼型与玩法，再套该风格材质

---

## 9. 验收检查表

1. 是否 **9:20** 竖屏手机框？
2. 楼型/玩法是否符合 §4 该行（不是统一锁定开火）？
3. 隐藏帮助能否走完**该行**核心循环？
4. 常驻交互是否认形（图标/仪表/指令），而非网页字钮？
5. 与相邻风格并排是否一眼可辨？
6. 文案是否中文且触控语义？无商业 Rip？

全部「是」才算完成。

---

## 10. 与风格 Spec 的关系

- `styles/*.md`：色、边、动效、反例 + **布局原型 / 核心玩法**（与 §4 一致）
- 本文：跨风格硬规则（9:20、触控、中文）+ 类型矩阵
- 风格可改材质与动效，**不得**推翻 9:20，也**不得**用统一动作条覆盖矩阵
