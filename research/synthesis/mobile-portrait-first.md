# 移动端竖屏优先（项目准则）

> 本仓库**为移动端竖屏游戏 UI 服务**。桌面浏览器仅作预览壳；验收与设计必须以竖屏触控为准。

## 1. 目标形态

| 项 | 约定 |
|----|------|
| 主画幅 | **竖屏**，默认参考比 **9:20**（亦兼容 9:16–9:19.5） |
| 主输入 | **触控**（点按、拖拽、长按、双指可选） |
| 预览 | 桌面用居中手机框模拟；**不得**把键鼠操作写成主操作说明 |
| 拇指区 | 关键操作落在屏幕下半区左右拇指可达范围 |

## 2. 必须遵守

1. **布局按竖屏信息架构**：顶状态 / 中视野 / 底操作栏；禁止把横屏四角 PC HUD 原样塞进竖屏。
2. **操作可单手或双手拇指完成**：开火、交互、换弹、菜单须有 **可见触控控件**（≥44×44 CSS px 热区，推荐 ≥48）。
3. **文案与提示用触控语义**：写「点按 / 拖动 / 长按」，不写「LMB / RMB / WASD / Tab」作为主路径。
4. **避免悬停依赖**：`:hover` 仅作增强；状态反馈以 `active` / `pressed` / 选中为准。
5. **安全区**：顶/底为刘海与手势条留白；关键数值勿贴物理边缘。
6. **方言/皮肤切换不得以键鼠为前提**：切换入口必须是屏幕上的按钮。
7. **Demo 验收**：真机或 DevTools 移动设备模式竖屏下，不看帮助文也能完成核心循环。
8. **文案语言**：所有玩家可见 UI 文本使用**中文**（按钮、HUD 标签、任务、提示、日志、菜单）。代码标识符可用英文；专有名词可括注原文。

## 3. 禁止（键鼠惯性）

- ❌ 把右键、中键、键盘快捷键当作**唯一**操作方式
- ❌ 帮助条主文案是键位表（`LMB FIRE · RMB LOCK · WASD`）
- ❌ 桌面全屏铺满当作默认交付形态（竖屏手机框才是默认）
- ❌ 控件热区过小、挤在屏幕上沿中央（竖屏难够）
- ❌ 依赖鼠标精确瞄准而不提供触控瞄准/锁定辅助

## 4. 推荐触控映射（动作 / RPG HUD 类）

| 意图 | 触控 |
|------|------|
| 环视 / 轻微视角 | 在世界区域 **拖动** |
| 选择目标 | **点按** 实体 |
| 开火 / 主操作 | 右下 **大键** |
| 交互 / 黑客 | 目标选中后的 **行动键** 或准星旁按钮 |
| 冲刺 | 左下 **按住** |
| 换弹 / 扫描 / 日志 | 底栏次级按钮 |
| 切武器 / 道具 | 底栏口袋格（已点按） |

键盘快捷键若保留，仅作桌面调试捷径，**不得**出现在玩家主帮助里。

## 5. 与风格文档的关系

- Spec 中的「几何与构图 / 可生成部件」须考虑竖屏锚点与触控尺寸。
- 各 `demo/scenes/*` 新做或大改时，默认交付 **9:20 竖屏 + 触控操作**。
- Style 01 参考实现：`demo/scenes/01-cyberpunk-hud/`（`layout-9x20.css` + 触控层）。

## 6. 一句话

> **先为拇指设计，再为预览兼容键鼠——不是反过来。**
