# audit-game-hud

对照本库硬规则与风格矩阵，对场景或用户项目中的游戏 HUD 做清单验收。

## 验收清单

对每一项给出 **通过** 或 **失败**，失败须附具体改法。

| # | 检查项 | 失败时怎么改 |
|---|--------|-------------|
| 1 | **画幅 9:20** — `#viewport` 或等效容器为竖屏 9:20 手机框 | 设 `aspect-ratio: 9 / 20`；用 phone-frame 壳，禁止横屏/全屏文档柱当交付 |
| 2 | **玩法可辨** — **有 pack**：楼型与核心循环符合矩阵该行；**通用路径**：符合该游戏自述的核心循环与类型正确楼型 | 对照矩阵或用户自述改布局；禁止换皮统一锁定开火 |
| 3 | **可见触控热区** — 主操作有 ≥44×44 CSS px 可见控件 | 加大可点区域；补图标键/物件键，勿仅靠 invisible hitbox |
| 4 | **中文文案** — 玩家可见 UI 为中文；`aria-label` 中文 | 替换英文主文案；专有名词可括注原文 |
| 5 | **无键鼠主路径** — 帮助与主流程不写 LMB/WASD/Tab；键鼠仅调试 | 改触控语义；调试快捷键移出玩家帮助 |
| 6 | **非网页壳** — 非居中卡片列表 + 双栏 `.btn`；控件同风格语汇 | 见 `docs/rules/game-feel-vs-web.md`：物件化主操作、消默认 button 皮 |
| 7 | **风格边界** — **有 pack**：与相邻风格并排可辨玩法差异；**通用**：不误套本库某 pack 专有视觉冒充「官方风格」 | pack 路径按矩阵还原独有楼型；通用路径保持自拟视觉一致、勿抄 pack 换皮 |
